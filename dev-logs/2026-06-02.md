# 开发日志 — 2026-06-02

## 完成事项

- [x] 修复平板/PC 端 `saveClip()` 异步竞态导致内容重复/乱序
  - 根因：`await this.db.addClip(clip)` 异步间隙，`lastContent` 被后续调用覆盖，去重失败
  - 修复：引入 `saveSeq` 序列号，await 后检查是否被新调用覆盖，过时结果丢弃
- [x] 修复多列网格外层 ForEach key 不稳定导致列重建
  - 根因：key 包含 `column.length`，增删后长度变化 → 约一半列整列销毁重建
  - 修复：key 改为固定 `'col_' + index`，列容器稳定，由内层 ForEach 处理差异
- [x] 修复搜索功能"完全搜不到"（200ms trailing-only 防抖）
  - 根因：trailing-edge 防抖要求停止输入 200ms 后才触发。快速连续输入时定时器不断重置，`searchKeyword` 和 `applyFilters()` 从未执行
  - 修复：改为 leading-edge + trailing debounce。首次击键立即搜索，后续击键在停止 200ms 后用最终关键词再搜索
  - 文件：ClipboardViewModel.ets `onSearch()`
- [x] 修复 SearchBar 状态与 ViewModel `searchKeyword` 解耦
  - 新增 `clearSearch()` 方法：清除防抖定时器 + 重置 searchKeyword + applyFilters
  - `deleteAll()` 调用 `clearSearch()` 确保清空操作后搜索状态完全重置
- [x] 修复 selectedClip 引用过期导致 ActionSheet 显示状态不准
  - 新增 `getSelectedClipActual()` 方法：通过 id 从当前 clips 数组查找最新引用
  - buildActionSheet 的置顶/取消置顶显示使用最新引用
- [x] 修复 Tab 切换后显示异常
  - 根因：Stack + Visibility 保活时，切回页面不会自动 rebuild。`@Trace` 在 Hidden 状态下更新可能未触发重绘
  - 修复：Index 新增 `tab0Refresh/tab1Refresh/tab2Refresh` 计数器，每个页面新增 `@Param refreshTrigger`。Tab 切换时递增对应计数器，强制目标页面 rebuild
- [x] 修复 `cleanExpired()` 在 init 时过早读取空 `fullClips` 的问题
  - 改为先过滤内存再异步清理 DB，配合 `loadClips()` 内部已有的过期过滤逻辑
- [x] 修复 `deleteAll()` 时序：先清内存再异步清 DB（乐观更新）
- [x] 修复初始断点 SM 布局闪烁：`aboutToAppear` 中主动获取窗口宽度初始化断点
- [x] 搜索大小写不敏感 + 图片参与搜索过滤
- [x] 搜索防抖 leading+trailing 模式
